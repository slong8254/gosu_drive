# 프로젝트 총괄 계획서

이 문서는 Gosu Drive 웹서비스 개발을 위한 전체 계획을 요약합니다. 초기 기획서의 내용을 기반으로 MVP 범위를 정의하고, 백엔드와 프런트엔드 구현 방향을 정리합니다.

## 1. 비전 및 목표
- 운전 연습 학원의 회원 관리와 수업 예약을 한 곳에서 처리하는 관리자 전용 백오피스를 구축합니다.
- 예약·취소 시 잔여시간을 정확히 차감·복구하여 시간 크레딧을 일관성 있게 관리합니다.
- 강사별 운영을 지원하며, 달력 기반의 빠른 예약 조회를 제공합니다.

## 2. 사용자와 권한
- **Owner**: 전체 설정과 관리자/강사 계정 관리.
- **Staff**: 회원·상품·예약 CRUD 및 잔여시간 조정.
- **Teacher**: 본인 관련 예약 조회(차후 제한적 수정 허용).

## 3. MVP 범위
- ### 회원 관리
  - 목록/검색/필터, 상세 프로필, 잔여시간 및 예약/결제 이력 조회
  - 시간 패키지 발행, 잔여시간 직접 조정(ledger 기록)
- ### 수업 상품
  - 강사, 기간, 기본 차감시간, 색상 라벨과 상태 관리
- ### 예약 처리
  - 회원 + 상품 + 시작/종료 시간 입력으로 예약 생성
  - 취소 시 소프트 삭제 및 100% 시간 복구
- ### 캘린더 뷰
  - FullCalendar로 월/주/일 조회, 드래그로 예약 생성
- ### 관리자 인증
  - Django Auth 기반 로그인 및 권한 제어

## 4. 데이터 모델 요약
- **Member**: 회원 정보와 `balance_minutes`
- **Teacher**: 사용자와 1:1 매핑, 강사 정보
- **Product**: 수업 상품, 담당 강사와 기간, 기본 차감시간
- **Booking**: 회원·상품·시간 범위·차감 분·상태·메모
- **TimeLedger**: 모든 시간 증감 기록(예약, 취소, 조정, 패키지)

## 5. 핵심 비즈니스 규칙
1. 예약 시 입력 분수 또는 상품 기본 차감시간으로 차감 분을 결정하고, 잔여시간 부족 시 경고 후 진행 여부 선택.
2. 예약 취소 시 booking.status='canceled'로 표시하지 않고 ledger(+분)로 100% 복구.
3. 잔여시간 조정은 관리 권한으로만 가능하며 ledger에 이유와 메모를 남김.
4. 모든 시간 변경은 트랜잭션과 `select_for_update`로 동시성 문제를 방지.

## 6. 프런트엔드 구성
- **기술**: Django Template, Tailwind CSS(Play CDN), Flowbite, HTMX, Alpine.js
- **기본 레이아웃**: 상단 네비게이션 바와 좌측 사이드바, 다크 모드 지원
- **주요 컴포넌트**: 예약 생성 모달, 잔여시간 조정 모달, 리스트 테이블, 캘린더 뷰
- **반응형**: 모바일 오프캔버스 사이드바, 데스크톱 2열 이상 레이아웃

## 7. API 및 서버 설계(요약)
- REST 스타일 엔드포인트: `/members/`, `/products/`, `/bookings/` 등
- 예약·취소·잔여시간 조정은 각각 전용 View와 Form에서 트랜잭션 처리
- 캘린더 피드는 `/calendar/feed/`에서 JSON 이벤트 제공

## 8. 테스트 전략
- 단위 테스트: 모델 비즈니스 로직(차감, 복구, 조정)
- 통합 테스트: 예약 생성/취소 API와 ledger 반영
- E2E 시나리오: 회원 생성 → 패키지 발행 → 예약 → 취소 → 잔액 검증

## 9. 로드맵
- **M0**: 모델 정의, 관리자 인증, 회원/상품/예약 기본 기능, 캘린더 피드
- **M1**: 출결, 리포트, 반복 예약
- **M2**: 강사 전용 뷰, 알림, 고급 분석 및 다지점 지원
